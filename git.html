Version control system - branching and mergin model

Git: Directed Acyclic Graph (DAG) on the repo

SVN: filesystem opration history from anywhere in the repo

Terminology

Working tree: the files visible in the filesystem

branch: pointer to a certain revision

HEAD: special branch: branch currently in the working tree

Staging: area where you group changes ready for next commit

Basic commands

status: check the current status of the repository

add: adds certain changes to staging environment

commit: adds the staged changes as one commit to HEAD

reset: remove local changes from staging (undo)

checkout: move HEAD to certain revision/branch

Distributed Revision Control System

Git: all repositories are peers, no master-slave

Centralised serve as Single Point Of Truth

Distributed commands

remote: other git repositories on other locations

Push: push a certain branch to a remote repo

Pull: the reverse: retrieve a certain branch from a remote repo

Fetch: get all changes from a remote

Advanced stuff

branching

Creating another line of revisions in the graph

Merging

Merging two branch lines back together so development work is integrated

Managed repositories

Repositories only used for managing activity such as CM

Pull requests

Requests to pull a branch, not vanilla Git but commonly used
